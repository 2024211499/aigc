# AIGC 教学助手工作流升级说明

## 设计依据

- Worked examples：解题类问题优先给“步骤示例”，不要只给结论。
- Spatial contiguity：图、公式、解释要放在同一张讲解卡附近，减少来回查找。
- Segmenting：移动端学习内容要按小节拆分，允许用户一段一段完成。
- Retrieval practice：学习流程必须包含测验和错题复盘，不能只生成讲义。

## 答疑意图分流

答疑入口先判断用户要什么，再决定输出形态。

| 意图 | 示例 | 输出 |
| --- | --- | --- |
| 词汇解释 | abandon | 词义、用法、例句、近义词 |
| 概览介绍 | 高数是学啥的 | 学科内容、用途、学习路线 |
| 解题推导 | 这道题怎么解 | 条件识别、步骤、公式块 |
| 明确画图 | 给我画受力图 | 图解、公式块、步骤 |
| 课程追问 | 这节课公式为什么这样写 | 结合当前讲义回答 |

原则：概览和词汇不出图、不出公式；只有明确解题、推导、公式、画图、受力、电路、几何、函数图像等场景才使用图和公式块。

## 课程模式闭环

课程模式保持四阶段：

1. 资料：上传教材、课件、论文。
2. 画像：采集基础、目标、节奏。
3. 路径：生成讲义、例题、练习、测验。
4. 闭环：根据测验和错题做复盘。

前端页面必须让用户知道当前处在哪一阶段，下一步是什么。

## 后端建议

Agent 输出建议统一增加 `answer_type` 字段：

- `vocabulary`
- `concept_overview`
- `problem_solving`
- `visual_explanation`
- `course_qna`

同时增加结构化字段：

- `summary`
- `steps`
- `formulas`
- `diagram_request`
- `followups`
- `practice_items`

前端不应从一段自然语言里猜公式和图，而应该优先读取这些结构化字段。
